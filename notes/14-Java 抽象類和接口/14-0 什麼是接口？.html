1. 什麼是 interface ？ -> 今天想開發一個海洋樂園遊戲，當中所有東西都會游泳。 「所有東西」都會「游泳」，而不是「某種東西」都會「游泳」。 「所有東西」都會「游泳」，代表了「游泳」這個「行為」可以被「所有東西」擁有，而不是「某種」東西專屬， 對於「定義行為」，在Java中可以使用interface關鍵字定義： // 1. 創建接口 // 修飾符 interface 接口名 { ... } public interface Swimmer { public abstract void swim(); } // 2. 父類使用接口 public abstract class Fish implements Swimmer { } // 3. 子類繼承父類並使用接口 public class 小丑魚(subtype) extends Fish { @Override public void swim() { System.out.printf("小丑魚 %s 游泳%n", name); } } 2. 3. 4. 5. 6. 6.
